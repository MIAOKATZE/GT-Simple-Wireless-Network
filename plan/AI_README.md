# GT-Simple-Wireless-Network 计划文档 (Plan.md) 编写规范

## 文档目的
本规范用于指导 `plan` 目录下所有 `plan_*.md` 文档的编写，确保内容结构清晰、易于追溯。

## 命名规则
所有计划文档必须遵循以下命名格式：
```
plan_<Tags_VERSION>_<YY-M-D>.md
```
- `<Tags_VERSION>`: 对应模组 `Tags.java` 文件中的版本号，例如 `0.0.2`。
- `<YY-M-D>`: 文档创建的日期，年份取两位（YY），月份（M）和日期（D）之间用短横线连接。例如 `26-4-7` 代表 2026年4月7日。

**示例**：
- `plan_0.0.2_26-4-7.md`

## 文档结构
标准的 Plan.md 应包含以下几个部分：
1.  **标题**: `# GT-Simple-Wireless-Network 改进计划 (Plan.md)`
2.  **摘要**: 简述本文档记录的主要内容。
3.  **分级列表**: 按 P0-P3 优先级列出问题。
4.  **状态追踪**: 每个问题项应包含其“状态”（如 ✅ 已修复、❌ 不予修改、⏸️ 暂缓、🔄 待讨论）和“处理说明”。
5.  **备注**: (可选) 记录生成者或其他相关信息。

## 更新原则
- 当问题被提出、解决、搁置或状态变更时，应及时更新对应条目的“状态”和“处理说明”。
- 确保文档内容与实际代码修改保持同步。
